About the Project

Credit Card Approval Prediction using Machine Learning - SmartBridge Virtual Internship

Project Context

In modern banking, credit card issuance represents a key source of revenue and customer acquisition. However, it also introduces significant financial risk if cards are approved for high-risk profiles. Traditionally, credit analysts reviewed applicants manually—evaluating incomes, educational titles, marital statuses, and job descriptions line-by-line.

This project automates the screening process using machine learning classifiers trained on real-world demographic structures and payment histories. By evaluating credit risk, banks can dramatically reduce bad debts, optimize human resource allocation, and grant instant credit cards to deserving applicants.

Dataset Details

We leverage two primary source csv datasets:

  • Application Records: Includes demographic and financial profiles (income, age, employment years, family size, car/property ownership).
  • Credit Records: Monthly tracking history of applicants' repayment behaviors (payment overlays and delinquent flags).

The target status code represents binary risk categories. Applicants with chronic late payments are classified as Rejected (1), while standard/low-risk clients are labeled as Approved (0).

Technical Pipeline
  • 1
    Clean: Dropping duplicate records, resolving empty strings, and handling missing occupation rows.
  • 2
    Engineering: Mapping raw date markers into positive numeric indicators (Age & Employed Years).
  • 3
    Aggregation: Summarizing monthly status indicators to compute maximum credit card repayment delays.
  • 4
    Transform: Executing StandardScaler scaling and LabelEncoder classifications.
  • 5
    Predict: Feeding scaled vectors to the best model classifier for eligibility assessment.

Core Advantages

  • Instant Inferences: Decision turnarounds reduced from 3 business days to milliseconds.
  • Risk Containment: Identifies delinquent behaviors early to protect capital assets.
  • Consistent Benchmarks: Eliminates human bias in customer profiles evaluations.
  • Scalability: Modular code structures support large application traffic.

Future Scope

  • Dynamic Tuning: Run automatic hyperparameter grid search optimization routines.
  • Alternative Data Ingestion: Connect social profile patterns and utility bill history.
  • IBM Cloud Migration: Deploy Flask as a microservice on Watson Machine Learning container environments.
  • Real-time API integrations: Allow retail e-commerce gateways to call the API directly.
